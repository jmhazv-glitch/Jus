# TEXTO COMPARATIVO 

Comparación entre Programación Tradicional y Programación Orientada a Objetos (POO)

La programación tradicional (o estructurada) se basa en el uso de funciones independientes que realizan tareas específicas. En este enfoque, los datos y las funciones están separados. Para este ejercicio, el programa tradicional utiliza funciones para ingresar temperaturas y calcular el promedio semanal, manteniendo una estructura simple y lineal.

Por otro lado, la Programación Orientada a Objetos organiza el código utilizando clases y objetos, donde los datos (atributos) y las funciones (métodos) están integrados en una misma entidad. Esto permite una mayor abstracción, reutilización y escalabilidad. En la versión POO del programa, se define una clase que encapsula las temperaturas, y se aplican principios como encapsulamiento e inclusión de herencia mediante una clase hija que muestra el promedio.

En general, la programación tradicional es adecuada para programas pequeños y directos, mientras que la POO es más útil en proyectos grandes y complejos donde se necesita organizar mejor el código y reutilizar componentes.
